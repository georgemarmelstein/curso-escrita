## Perfis de Saída

### Perfil: Operador Básico

**Pré-requisito:** Módulo 1, Aulas 0-2

**Competências:**
- Instalar e configurar Claude Code
- Usar interface de linha de comando
- Criar CLAUDE.md básico
- Executar tarefas simples

**Resultado:** Capaz de usar Claude Code para tarefas individuais

### Perfil: Construtor de Agentes

**Pré-requisito:** Módulo 1 completo + Aula 6

**Competências:**
- Todas do Operador Básico
- Criar Commands e Skills
- Configurar MCPs
- Projetar pipelines simples

**Resultado:** Capaz de construir ferramentas customizadas

### Perfil: Arquiteto de Sistemas

**Pré-requisito:** Módulos 1 e 2 completos

**Competências:**
- Todas do Construtor de Agentes
- Projetar sistemas completos
- Orquestrar múltiplos agentes
- Implementar governança

**Resultado:** Capaz de projetar sistemas agênticos complexos

### Perfil: Regente Cognitivo

**Pré-requisito:** Curso completo + prática deliberada

**Competências:**
- Todas anteriores
- Avaliar criticamente outputs
- Decidir níveis de autonomia
- Supervisionar sistemas em produção
- Ensinar outros

**Resultado:** Profissional transformado, multiplicador de capacidade

## Avaliação de Competências

### Indicadores de Domínio

| Nível | Descrição | Indicador |
|-------|-----------|-----------|
| **Iniciante** | Precisa de instruções passo a passo | Segue tutorial |
| **Praticante** | Executa com autonomia supervisionada | Resolve variações |
| **Competente** | Executa com autonomia total | Adapta a novos contextos |
| **Proficiente** | Otimiza e ensina outros | Identifica melhorias |
| **Expert** | Inova e cria novos padrões | Contribui para comunidade |

### Exercícios de Validação

Cada competência tem exercício associado que valida domínio:

| Competência | Exercício de Validação |
|-------------|----------------------|
| T1 (Instalar) | Instalação funcional verificada |
| T2 (CLAUDE.md) | Projeto configurado corretamente |
| T4 (Skills) | Skill funcional criada |
| A1 (Pipelines) | Pipeline executando end-to-end |
| A6 (Sistema) | CLERK funcional para caso real |

## Pré-requisitos de Entrada

### Conhecimentos Prévios Necessários

| Requisito | Nível | Onde Adquirir |
|-----------|-------|---------------|
| Uso de terminal/CLI | Básico | Kit de instalação |
| Conceitos de IA | Familiaridade | Aula 0 |
| Lógica de programação | Conceitual | Não exigido |
| Domínio jurídico | Praticante | Background do aluno |

### Atitudes Necessárias

1. **Disposição para experimentar:** Terminal pode intimidar inicialmente
2. **Tolerância à ambiguidade:** Sistemas agênticos são estocásticos
3. **Pensamento crítico:** Validar outputs é responsabilidade humana
4. **Persistência:** Prompt Learning Loop exige iteração

## Certificação de Competências

### Critérios para Conclusão

- [ ] Completar todos os exercícios das aulas
- [ ] Construir pelo menos 1 Command funcional
- [ ] Construir pelo menos 1 Skill funcional
- [ ] Executar pipeline CLERK em caso real
- [ ] Demonstrar revisão crítica de output

### Níveis de Certificação

| Nível | Requisito | Competências Validadas |
|-------|-----------|----------------------|
| **Bronze** | Módulo 1 completo | C1-C7, T1-T7 |
| **Prata** | Módulos 1+2 completos | Todas + P1-P5, X1-X5 |
| **Ouro** | Todos + projeto aplicado | Todas + demonstração prática |

## Aplicação no Domínio Jurídico

### Competências Jurídicas Específicas

| Competência | Aplicação Jurídica |
|-------------|-------------------|
| Pipeline design | Fluxo processual automatizado |
| Gates e checkpoints | Validação de peças |
| MCPs jurídicos | Pesquisa de precedentes |
| Revisão com subagentes | Controle de qualidade de minutas |
| Governança | Rastreabilidade de decisões |

### Casos de Uso Validados

1. **Triagem de processos:** Classificação automática por tema
2. **Pesquisa de precedentes:** BNP, CJF, JULIA
3. **Extração de fatos:** Relatórios estruturados
4. **Elaboração de minutas:** Com revisão multi-agente
5. **Verificação de consistência:** Confronto com autos
