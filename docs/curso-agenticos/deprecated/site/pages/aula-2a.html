O Modelo Fundamental

~10 minutos

ENTRADA - PROCESSAMENTO - SAIDA

O insight mais simples e mais poderoso: Todo sistema agêntico, por mais complexo que seja, segue este modelo. A diferença está na QUALIDADE de cada etapa. 

┌─────────────────────────────────────────────────────────────────────────┐
│                                                                         │
│    ENTRADA           PROCESSAMENTO           SAÍDA                      │
│    ───────           ─────────────           ─────                      │
│                                                                         │
│    • Prompt          • Raciocínio            • Resposta                 │
│    • Contexto        • Uso de ferramentas    • Arquivo                  │
│    • Arquivos        • Consultas             • Ação                     │
│    • Histórico       • Decisões              • Código                   │
│                                                                         │
│         │                   │                     │                     │
│         ▼                   ▼                     ▼                     │
│    ┌─────────┐        ┌──────────┐         ┌──────────┐                │
│    │ VOCÊ    │   →    │ AGENTE   │    →    │ RESULTADO│                │
│    │ decide  │        │ executa  │         │ entregue │                │
│    └─────────┘        └──────────┘         └──────────┘                │

Os 3 Pilares da Autonomia

Michele Catasta (Stanford/Google) identifica três pilares que determinam o nível de autonomia de um agente: 

┌─────────────────────────────────────────────────────────────────────────┐
│                    3 PILARES DA AUTONOMIA                               │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   ┌─────────────┐   ┌─────────────┐   ┌─────────────┐                  │
│   │ CAPABILITY  │   │ VERIFICATION│   │   CONTEXT   │                  │
│   │ Capacidade  │   │ Verificação │   │  Contexto   │                  │
│   └──────┬──────┘   └──────┬──────┘   └──────┬──────┘                  │
│          │                 │                 │                          │
│          ▼                 ▼                 ▼                          │
│    O que o agente    Como garantir     Informações que                 │
│    SABE fazer        que fez CERTO     o agente PRECISA                │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘

Aplicação Prática

  • Capability: Quais ferramentas o agente tem? Quais tarefas sabe executar?
  • Verification: Como verificamos se a saída está correta? Há revisão?
  • Context: O agente tem todas as informações necessárias?
Bloco 02

Níveis de Autonomia (L1-L5)

~10 minutos

A Escala L1 a L5

Inspirada em carros autônomos, esta escala ajuda a entender onde seu agente opera: 

┌─────────────────────────────────────────────────────────────────────────┐
│                     ESCALA DE AUTONOMIA                                 │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│  L1 │ OPERATOR      │ Humano faz tudo, IA sugere                       │
│     │ (Operador)    │ "O que você acha dessa ideia?"                   │
│─────┼───────────────┼───────────────────────────────────────────────── │
│  L2 │ REVIEWER      │ IA faz, humano revisa TUDO                       │
│     │ (Revisor)     │ "Aqui está o rascunho, aprove."                  │
│─────┼───────────────┼───────────────────────────────────────────────── │
│  L3 │ APPROVER      │ IA faz, humano aprova DECISÕES-CHAVE             │
│     │ (Aprovador)   │ "Posso enviar esse e-mail?"                      │
│─────┼───────────────┼───────────────────────────────────────────────── │
│  L4 │ AUDITOR       │ IA faz, humano audita RESULTADOS                 │
│     │ (Auditor)     │ "Processou 47 casos, veja o resumo."             │
│─────┼───────────────┼───────────────────────────────────────────────── │
│  L5 │ OBSERVER      │ IA faz tudo, humano OBSERVA                      │
│     │ (Observador)  │ "Sistema rodando, métricas normais."             │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘

Onde Cada Nível Faz Sentido

Nível Caso de Uso Jurídico Risco
L1 Tirar dúvidas de legislação Baixo
L2 Redigir minutas para revisão Médio
L3 Pipeline de relatórios com aprovação Médio
L4 Triagem automática de processos Alto
L5 Monitoramento de prazos Baixo (se bem configurado)

Princípio Fundamental

Quanto maior o impacto da decisão, menor deve ser a autonomia do agente.

O Paradoxo da Confiança

┌─────────────────────────────────────────────────────────────────────────┐
│                  PARADOXO DA CONFIANÇA                                  │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   "Para confiar mais no agente, você precisa                           │

Zona Inteligente vs Zona de Execução

Conceito crucial para entender onde o LLM agrega valor:

┌─────────────────────────────────────────────────────────────────────────┐
│           ZONA INTELIGENTE vs ZONA DE EXECUÇÃO                          │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   ZONA INTELIGENTE              ZONA DE EXECUÇÃO                        │
│   (LLM decide)                  (LLM executa)                           │
│   ────────────────              ──────────────────                      │
│                                                                         │
│   • Qual estratégia usar?       • Ler arquivo X                         │
│   • Quais ferramentas chamar?   • Buscar no banco Y                     │
│   • Como estruturar a resposta? • Escrever arquivo Z                    │
│   • Quando pedir ajuda?         • Executar comando W                    │
│                                                                         │
│   ┌─────────────────┐           ┌─────────────────┐                    │
│   │   CRIATIVIDADE  │           │  DETERMINISMO   │                    │
│   │   JULGAMENTO    │           │  PRECISÃO       │                    │
│   │   SÍNTESE       │           │  REPETIÇÃO      │                    │
│   └─────────────────┘           └─────────────────┘                    │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘

Insight Prático

Deixe o LLM pensar na zona inteligente. Deixe ferramentas determinísticas executar na zona de execução.

O Padrão RPI (Research-Plan-Implement)

Padrão identificado em agentes de alta performance:

┌─────────────────────────────────────────────────────────────────────────┐
│                    PADRÃO RPI                                           │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   R ─ RESEARCH (Pesquisar)                                              │
│       │                                                                 │
│       │  "Antes de agir, entender"                                      │
│       │  • Ler documentos relevantes                                    │
│       │  • Buscar precedentes                                           │
│       │  • Mapear contexto                                              │
│       │                                                                 │
│       ▼                                                                 │
│   P ─ PLAN (Planejar)                                                   │
│       │                                                                 │
│       │  "Antes de executar, planejar"                                  │
│       │  • Definir etapas                                               │
│       │  • Identificar dependências                                     │
│       │  • Antecipar problemas                                          │
│       │                                                                 │
│       ▼                                                                 │
│   I ─ IMPLEMENT (Implementar)                                           │
│                                                                         │
│       "Executar o plano"                                                │
│       • Seguir etapas definidas                                         │
│       • Verificar cada resultado                                        │
│       • Ajustar se necessário                                           │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘
RPI Equivalente Jurídico
Research Ler os autos, buscar jurisprudência
Plan Definir estrutura da sentença
Implement Redigir a decisão

Os 12 Fatores de um Agente Bem Projetado

Dex Horthy (Prefect) adaptou os "12 Factor Apps" para agentes:

# Fator Significado
1 Prompts são código Versionados, testados, revisados
2 Context is King Contexto rico > instruções longas
3 Ferramentas simples Cada ferramenta faz UMA coisa bem
4 Ferramentas unificadas Interface consistente
5 Crie, não oriente Estruturar > instruir
6 Trate como junior Instruções explícitas
7 Outcomes, not operations Defina resultados, não passos
8 Contexto isolado Cada tarefa com seu escopo
9 Falhe graciosamente Recuperação de erros
10 Log tudo Rastreabilidade
11 Pequenas iterações Evite mega-planos
12 Controle o desvio Limites claros

O Que São Ferramentas (Tools)

Ferramentas são a INTERFACE do agente com o mundo. 

┌─────────────────────────────────────────────────────────────────────────┐
│                       FERRAMENTAS                                       │
├─────────────────────────────────────────────────────────────────────────┤
│                                                                         │
│   ┌─────────────┐    ┌─────────────┐    ┌─────────────┐                │
│   │    LER      │    │   BUSCAR    │    │  EXECUTAR   │                │
│   │  arquivos   │    │ informação  │    │   ações     │                │
│   └─────────────┘    └─────────────┘    └─────────────┘                │
│                                                                         │
│   Exemplos:                                                             │
│   • Read (ler arquivo)                                                  │
│   • Write (escrever arquivo)                                            │
│   • Bash (executar comando)                                             │
│   • WebSearch (buscar na web)                                           │
│   • MCP Tools (ferramentas externas)                                    │
│                                                                         │
└─────────────────────────────────────────────────────────────────────────┘

A Regra dos 50

Jeremiah Lowin (Prefect) descobriu um limite prático:

JL

"Acima de 50 ferramentas por agente, a performance degrada significativamente."

Jeremiah Lowin

Prefect

Implicação

  • Não dê TODAS as ferramentas para UM agente
  • Especialize agentes com conjuntos menores de ferramentas
  • Use subagentes para tarefas específicas

Outcomes, Not Operations

Princípio crucial para design de ferramentas:

┌─────────────────────────────────────────────────────────────────────────┐
│              OUTCOMES, NOT OPERATIONS                                   │

│      • read_line()                                                      │
│      • close_file()                                                     │
│                                                                         │

│                                                                         │
│   ────────────────────────────────────────────────────────────────      │
│                                                                         │
│   ❌ ERRADO: "Abra o arquivo, leia linha por linha..."                  │

Padrões de Supervisão Humana

Padrão Descrição Uso
HITL Human-in-the-loop (dentro do loop) Decisões críticas, ações irreversíveis
HOTL Human-on-the-loop (monitora, pode intervir) Tarefas de baixo risco, alto volume
HOOTL Human-out-of-the-loop (só vê resultado final) Automações maduras, bem testadas

Vibe Engineering vs Vibe Coding

Vibe Coding (amador) Vibe Engineering (profissional)
"Deixa a IA fazer" "Eu projeto, IA executa"
Aceita primeira resposta Itera até qualidade
Não entende o output Entende cada linha
Não testa Testa rigorosamente
Não documenta Documenta decisões
