Contexto e Rei

Objetivo

Dominar a arte de gerenciar o contexto que alimenta sistemas agenticos. Estruturar informacoes de forma que o agente tenha acesso ao que precisa, quando precisa, sem desperdicar tokens.

"Context is King. A well-structured context makes the model smarter than any instruction could."

Dex Horthy (12 Factor Agents)

Lema da Aula

"Contexto rico supera instrucoes longas."

Pre-requisitos

  • Aula 6: Engenharia de Prompt
  • Compreensao de CLAUDE.md
  • Experiencia basica com tokens e custos

Bloco 1: O Problema do Contexto Inchado

20 min

1.1 Por Que CLAUDE.md Gigante e Anti-Padrao

"Metadata visible, details loaded on demand."

Barry Zhang, Anthropic (AI Engineer Summit 2024) 
┌─────────────────────────────────────────────────────────────────┐
│              O ANTI-PADRAO: CLAUDE.md GIGANTE                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   SINTOMAS:                                                     │
│   ─────────                                                     │
│   • CLAUDE.md com mais de 500 linhas                            │
│   • Documentacao inteira embutida                               │
│   • Exemplos longos inline                                      │

│   • Logs de reunioes                                            │
│                                                                 │

│   3. QUALIDADE: Modelo pode se confundir com excesso            │
│   4. MANUTENCAO: Dificil manter atualizado                      │
│   5. DESATUALIZACAO: Copias ficam defasadas                     │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

Bloco 3: Hierarquia de CLAUDE.md

25 min

3.1 Os 3 Niveis

┌─────────────────────────────────────────────────────────────────┐
│              HIERARQUIA DE CLAUDE.md                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   NIVEL 1: GLOBAL (~/.claude/CLAUDE.md)                         │
│   ─────────────────────────────────────                         │
│   Configuracoes que valem para TODOS os projetos:               │
│   • Preferencias de idioma                                      │
│   • Estilo de comunicacao                                       │
│   • Ferramentas globais instaladas                              │
│   • Regras pessoais do usuario                                  │
│                                                                 │
│   NIVEL 2: PROJETO (./CLAUDE.md)                                │
│   ────────────────────────────────                              │
│   Configuracoes especificas do projeto:                         │
│   • Arquitetura do sistema                                      │
│   • Padroes de codigo                                           │
│   • Dependencias e stack                                        │
│   • Commands e Skills do projeto                                │
│                                                                 │
│   NIVEL 3: SUBPASTA (./pasta/CLAUDE.md)                         │
│   ─────────────────────────────────────                         │
│   Configuracoes especificas de uma pasta:                       │
│   • Contexto local (ex: tests/, docs/)                          │
│   • Regras especificas (ex: sem console.log)                    │
│   • Sobrescreve niveis superiores                               │
│                                                                 │
│   ┌─────────────────────────────────────────────────────────┐   │
│   │                                                         │   │
│   │   ~/.claude/CLAUDE.md     (GLOBAL - sempre carregado)   │   │
│   │           │                                             │   │
│   │           ▼                                             │   │
│   │   ./CLAUDE.md             (PROJETO - no projeto atual)  │   │
│   │           │                                             │   │
│   │           ▼                                             │   │
│   │   ./tests/CLAUDE.md       (SUBPASTA - contexto local)   │   │
│   │                                                         │   │
│   └─────────────────────────────────────────────────────────┘   │
│                                                                 │
│   ORDEM DE PRECEDENCIA:                                         │
│   Subpasta > Projeto > Global                                   │
│   (mais especifico sobrescreve mais generico)                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

│   A Context Window e o "espaco de memoria" disponivel           │

4.2 Comando /compact

O comando /compact comprime o historico da conversa, liberando espaco na context window.

Quando Usar

  • Conversa muito longa (100+ mensagens)
  • Respostas ficando lentas ou truncadas
  • Aviso de context window cheia
  • Mudanca de tarefa dentro da mesma sessao

Quando Iniciar Nova Sessao

  • Tarefa completamente diferente
  • Contexto antigo nao e mais relevante
  • Apos muitos erros acumulados
  • Performance muito degradada

Bloco 5: Context is King

20 min

5.1 Contexto Rico vs Instrucoes Longas

"Um contexto bem estruturado torna o modelo mais inteligente do que qualquer instrucao poderia."

Dex Horthy (12 Factor Agents) 
┌─────────────────────────────────────────────────────────────────┐
│              CONTEXTO RICO vs INSTRUCOES LONGAS                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ABORDAGEM ERRADA (Instrucoes Longas):                         │
│   ─────────────────────────────────────                         │
│   "Gere uma sentenca. Uma sentenca tem relatorio,               │
│   fundamentacao e dispositivo. O relatorio deve conter          │
│   os fatos, as partes, os pedidos. A fundamentacao              │
│   deve analisar cada pedido. O dispositivo deve..."             │
│   [500 palavras de instrucoes]                                  │
│                                                                 │
│   ABORDAGEM CORRETA (Contexto Rico):                            │
│   ──────────────────────────────────                            │
│   "Gere sentenca seguindo o template em:                        │
│   docs/template-sentenca-bpc.md                                 │
│                                                                 │
│   Contexto do caso:                                             │
│   - Tipo: BPC Idoso                                             │
│   - Requerente: [dados]                                         │
│   - Laudo: Favoravel                                            │
│   - Precedente aplicavel: Tema 1038 STF"                        │
│                                                                 │
│   A DIFERENCA:                                                  │
│   ─────────────                                                 │
│   • Instrucoes longas: Agente precisa "decorar" regras          │
│   • Contexto rico: Agente tem EXEMPLOS e DADOS                  │
│                                                                 │
│   Por que contexto funciona melhor:                             │
│   • LLMs aprendem por EXEMPLOS, nao por regras                  │
│   • Template concreto > descricao abstrata                      │
│   • Dados reais > instrucoes hipoteticas                        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

5.2 Estrutura Ideal

Elemento O que incluir Formato
Identidade Quem voce e, qual o projeto 2-3 linhas
Estrutura Pastas, arquivos importantes Lista ou arvore
Padroes Convencoes do projeto Tabela ou lista
Referencias Onde encontrar detalhes Links para arquivos

Resumo da Aula

Conceitos-Chave

  • Progressive Disclosure: Metadados visiveis, detalhes sob demanda
  • Hierarquia: Global > Projeto > Subpasta
  • CLAUDE.md Ideal: 200-300 linhas
  • Context Window: Espaco finito, gerenciar com /compact
  • Contexto Rico: Supera instrucoes longas

Especialistas Citados
