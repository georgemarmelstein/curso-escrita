Capacidades Encapsuladas

Objetivo

Aprender a criar Commands e Skills - blocos de expertise que o agente pode usar repetidamente com consistencia e qualidade.

"Skills are just folders with a SKILL.md file."

Barry Zhang, Anthropic (AI Engineer Summit 2024)

Bloco 1: Commands - Atalhos de Prompt

25 min

1.1 O Que Sao Commands

Commands sao prompts salvos que voce invoca com o prefixo /. Transformam instrucoes repetitivas em comandos simples.

┌─────────────────────────────────────────────────────────────────┐
│                       O QUE SAO COMMANDS                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   SEM COMMAND:                                                  │
│   ─────────────                                                 │
│   "Analise este documento juridico e crie um resumo com fatos,  │
│    argumentos e conclusoes. Use formato Markdown com secoes     │
│    numeradas. Cite a pagina fonte de cada informacao..."        │
│                                                                 │
│   (digitado toda vez, tokens gastos repetidamente)              │
│                                                                 │
│   COM COMMAND:                                                  │
│   ─────────────                                                 │
│   /resumir-processo documento.pdf                               │
│                                                                 │
│   (mesmo resultado, invocacao simples)                          │
│                                                                 │
│   BENEFICIOS:                                                   │
│   • Economia de tokens (prompt nao repetido na conversa)        │
│   • Consistencia (mesmo prompt toda vez)                        │
│   • Velocidade (menos digitacao)                                │
│   • Compartilhamento (equipe usa os mesmos commands)            │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

1.2 Onde Ficam os Commands

┌─────────────────────────────────────────────────────────────────┐
│                  LOCALIZACAO DOS COMMANDS                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   COMMANDS GLOBAIS (disponiveis em qualquer projeto):           │
│   ~/.claude/commands/                                           │
│       ├── resumir-documento.md        → /resumir-documento      │
│       ├── pesquisar-bnp.md            → /pesquisar-bnp          │
│       └── traduzir-ingles.md          → /traduzir-ingles        │
│                                                                 │
│   COMMANDS DO PROJETO (so neste projeto):                       │
│   ./projeto/.claude/commands/                                   │
│       ├── relatar-processo.md         → /relatar-processo       │
│       ├── analisar-caso.md            → /analisar-caso          │
│       └── elaborar-minuta.md          → /elaborar-minuta        │
│                                                                 │
│   REGRA DE NOMENCLATURA:                                        │
│   • Nome do arquivo = nome do comando (sem extensao)            │
│   • Use kebab-case (palavras-separadas-por-hifen)               │
│   • Evite caracteres especiais ou espacos                       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

1.3 Variaveis de Command

Variavel O que captura Exemplo de uso
$ARGUMENTS Tudo apos o comando /resumir documento.pdf
$FILE Arquivo selecionado no IDE Arquivo aberto no VS Code

Exemplo de Command

# resumir-documento.md

Analise o documento $ARGUMENTS e gere um resumo estruturado.

## Formato de Saida
1. FATOS: Lista cronologica
2. PARTES: Identificacao completa
3. PEDIDOS: Itemizacao
4. FUNDAMENTOS: Base legal

SEMPRE citar pagina de referencia.
NUNCA omitir valores monetarios.

Bloco 2: Anatomia da Skill

30 min

2.1 O Que Sao Skills

"Skills are just folders with a SKILL.md file."

Barry Zhang, Anthropic

Skills sao pastas com expertise encapsulada. O Claude detecta automaticamente e carrega quando relevante.

┌─────────────────────────────────────────────────────────────────┐
│                  ESTRUTURA DE UMA SKILL                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ~/.claude/skills/                                             │
│       └── relatorio-judicial/          ← Nome da skill          │
│           └── SKILL.md                 ← Arquivo obrigatorio    │
│                                                                 │
│   CONTEUDO DO SKILL.md:                                         │
│   ─────────────────────                                         │
│   # Relatorio Judicial                  ← Nome da skill         │
│                                                                 │
│   ## Quando Usar                        ← Trigger conditions    │
│   Quando o usuario pedir para gerar relatorio de processo       │
│   judicial ou analisar documentos processuais.                  │
│                                                                 │
│   ## Como Funciona                      ← Metodologia           │
│   1. Le o documento inteiro                                     │
│   2. Extrai fatos cronologicamente                              │
│   3. Identifica partes e pedidos                                │
│   4. Mapeia fundamentos legais                                  │
│                                                                 │
│   ## Regras                             ← SEMPRE/NUNCA/PREFERIR │
│   SEMPRE incluir referencia de pagina                           │
│   NUNCA omitir valores monetarios                               │
│   PREFERIR citacao direta a parafraseamento                     │
│                                                                 │
│   ## Formato de Saida                   ← Template              │
│   ```                                                           │
│   # RELATORIO                                                   │
│   ## FATOS                                                      │
│   ## PARTES                                                     │
│   ## PEDIDOS                                                    │
│   ## FUNDAMENTOS                                                │
│   ```                                                           │
│                                                                 │
│   ## Exemplos                           ← Entrada → Saida       │
│   [Exemplos concretos]                                          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

2.2 Secoes Obrigatorias

Secao Funcao Conteudo
Quando Usar Trigger conditions Quando a skill deve ser ativada
Como Funciona Metodologia Passos que o agente deve seguir
Regras Restricoes SEMPRE/NUNCA/PREFERIR
Formato Template Estrutura esperada da saida
Exemplos Demonstracao Entrada → Saida esperada

Bloco 3: Command vs Skill - Quando Usar Cada

20 min

3.1 Regra Pratica

Regra de 50 Linhas

Se o prompt tem menos de 50 linhas e nao precisa de exemplos complexos, use Command. Caso contrario, use Skill.

3.2 Arvore de Decisao

┌─────────────────────────────────────────────────────────────────┐
│              COMMAND vs SKILL - ARVORE DE DECISAO               │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   Pergunta 1: E invocado EXPLICITAMENTE pelo usuario?           │
│   ├── SIM → Provavelmente COMMAND                               │
│   └── NAO → Pergunta 2                                          │
│                                                                 │
│   Pergunta 2: Precisa de EXEMPLOS complexos (entrada→saida)?    │
│   ├── SIM → SKILL                                               │
│   └── NAO → Pergunta 3                                          │
│                                                                 │
│   Pergunta 3: Tem MAIS de 50 linhas de instrucoes?              │
│   ├── SIM → SKILL                                               │
│   └── NAO → Pergunta 4                                          │
│                                                                 │
│   Pergunta 4: Deve ser ativado por CONTEXTO automaticamente?    │
│   ├── SIM → SKILL                                               │
│   └── NAO → COMMAND                                             │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

3.3 Comparacao

Aspecto Command Skill
Invocacao Explicita (/comando) Automatica (por contexto)
Tamanho < 50 linhas Ilimitado
Exemplos Simples ou nenhum Complexos, multiplos
Estrutura Arquivo unico .md Pasta com SKILL.md
Uso tipico Atalhos rapidos Expertise especializada

Bloco 4: Templates e Exemplos

25 min

4.1 Por Que Exemplos Sao Essenciais

"LLMs aprendem melhor por exemplos do que por regras. Um bom exemplo vale mais que 100 linhas de instrucoes."

┌─────────────────────────────────────────────────────────────────┐
│              POR QUE EXEMPLOS FUNCIONAM                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   SEM EXEMPLO:                                                  │
│   ─────────────                                                 │
│   "Gere um relatorio com fatos, partes e pedidos"               │
│   → Agente INTERPRETA o formato                                 │
│   → Resultado INCONSISTENTE                                     │
│                                                                 │
│   COM EXEMPLO:                                                  │
│   ─────────────                                                 │
│   "Gere um relatorio seguindo este modelo:                      │
│                                                                 │
│   # RELATORIO DO PROCESSO 0000001-00.2024.5.05.0001             │
│                                                                 │
│   ## FATOS                                                      │
│   1. Em 15/01/2024, o autor ajuizou acao (fl. 1-10)             │
│   2. Citacao realizada em 20/01/2024 (fl. 15)                   │
│                                                                 │
│   ## PARTES                                                     │
│   - AUTOR: Joao da Silva, CPF xxx.xxx.xxx-xx                    │
│   - REU: INSS"                                                  │
│                                                                 │
│   → Agente COPIA a estrutura                                    │
│   → Resultado CONSISTENTE                                       │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘

4.2 Estrutura de Exemplos

Todo exemplo deve ter:

  • Entrada: O que o usuario fornece
  • Contexto: Informacoes relevantes
  • Saida: Resultado esperado exato

4.3 Boas Praticas

Pratica Por que
Incluir caso simples E complexo Mostra range de aplicacao
Mostrar edge cases Previne erros em situacoes atipicas
Usar dados realistas Mais facil de generalizar
Anotar o raciocinio Explica o "por que" das escolhas

5.2 Convencoes de Nomenclatura

Tipo Convencao Exemplo
Pastas kebab-case relatorio-judicial/
Commands kebab-case analisar-caso.md
Skills kebab-case (pasta) elaboracao-minuta/
Arquivo skill SKILL.md (fixo) SKILL.md

Resumo da Aula

Conceitos-Chave

  • Command: Prompt salvo invocado com / (< 50 linhas)
  • Skill: Expertise em pasta com SKILL.md
  • $ARGUMENTS: Captura texto apos o comando
  • Regra 50 linhas: Criterio para Command vs Skill
  • Exemplos: Entrada → Saida (essenciais para Skills)
