# ROTEIRO DE SUPERAULA

## Metadados
- **Tema:** Aula 1.1 — O Manifesto SuperJuristas
- **Duração estimada:** 35-40 minutos
- **Público-alvo:** Juristas que completaram a Aula 0 (sensibilização)
- **Pré-requisitos:** Aula 0 — Bem-Vindo ao Mundo Agêntico
- **Ambiente:** Apresentação + reflexão guiada
- **Estrutura narrativa:** Declaração de Princípios

---

## Propósito

Na Aula 0, você aceitou o chamado à aventura. Viu as limitações do assistente, conheceu a promessa do agente, entendeu os desafios do caminho.

Agora, antes de dar o primeiro passo prático, precisamos estabelecer algo mais fundamental: **quem você está se tornando** e **por que isso importa**.

Esta aula não ensina ferramentas. Ensina **filosofia operacional**.

Porque a diferença entre quem usa agentes como brinquedo e quem os usa como instrumento de transformação não está na técnica — está na **visão de mundo**.

Este é o Manifesto SuperJuristas. E você não é apenas leitor — é signatário.

---

## Nota sobre Terminologia

> **Metáforas vs Conceitos Técnicos**
>
> Neste curso, usamos termos como "Regente Cognitivo" e "Orquestra Cognitiva" como **metáforas** para tornar os conceitos memoráveis e acessíveis.
>
> Os termos técnicos oficiais da Anthropic são: **Agent**, **Skill**, **Tool Use**, **Task Coordination**, **Context Window**.
>
> Quando a distinção importar, usaremos notas técnicas para esclarecer.

# PARTE 1: MISSÃO E VISÃO

---

### Missão SuperJuristas (2 min)

```
┌─────────────────────────────────────────────────────────────────────┐
│                                                                     │
│   MISSÃO SUPERJURISTAS                                              │
│                                                                     │
│   Formar REGENTES COGNITIVOS — profissionais do Direito que        │
│   dominam a arte de projetar, construir e supervisionar            │
│   sistemas agênticos para amplificar sua capacidade de             │
│   análise, decisão e produção de justiça.                          │
│                                                                     │
│   ─────────────────────────────────────────────────────────────    │
│                                                                     │
│   Não ensinamos a apertar botões. Ensinamos a conceber sinfonias.  │
│   Não treinamos operadores. Formamos arquitetos de cognição.       │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

**Palavras-chave:**
- **Regentes Cognitivos** — não operadores, não usuários
- **Projetar, construir, supervisionar** — não apenas usar
- **Amplificar** — não substituir
- **Produção de justiça** — o propósito final

---

### Visão SuperJuristas (3 min)

```
┌─────────────────────────────────────────────────────────────────────┐
│                                                                     │
│   VISÃO SUPERJURISTAS                                               │
│                                                                     │
│   Um sistema de justiça onde cada profissional tem à disposição    │
│   uma ORQUESTRA COGNITIVA PERSONALIZADA — agentes que              │
│   potencializam sua capacidade humana sem substituí-la.            │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

**Um futuro onde:**
- Nenhum processo prescreve por falta de tempo

**Reflexão guiada:**
> "Pense no seu último mês. Quantos processos você analisou com a profundidade que mereciam? Quantos você teve que 'dar conta' porque não havia tempo?"

> "A visão SuperJuristas não é utopia. É consequência direta do que estamos construindo."

**Transição:**

# PARTE 2: OS 10 PRINCÍPIOS

---

### Bloco 1: O Papel do Regente — Princípios I-IV (8 min)

> "Os quatro primeiros princípios definem **quem você é** neste novo mundo."

---

**Princípio I: Do Operacional ao Estratégico**

```
┌─────────────────────────────────────────────────────────────────────┐
│  PRINCÍPIO I                                                        │
│                                                                     │
│         Os agentes assumem: pesquisar, redigir, organizar,         │
│         converter, comparar, verificar, cruzar, sintetizar.        │
│                                                                     │
│         Você concentra-se em: DECIDIR, VALORAR, JULGAR.            │
│                                                                     │
│   ─────────────────────────────────────────────────────────────    │
│                                                                     │
│         Estratégico não significa distante.                         │
│         Significa ESSENCIAL.                                        │
│         Presente onde a presença humana é insubstituível.           │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

**Exemplo jurídico:**
- **Operacional (agente):** Pesquisar 500 precedentes sobre "auxílio-doença negado por alta programada"
- **Estratégico (você):** Decidir qual tese jurídica aplicar a este caso específico

> O agente pesquisa. Você escolhe a estratégia.

---

**Princípio II: O Curador de Três Domínios**

```
┌─────────────────────────────────────────────────────────────────────┐
│  PRINCÍPIO II                                                       │
│                                                                     │
│         ┌─────────────┬─────────────┬─────────────────┐            │
│         │ CONHECIMENTO│   VALORES   │    CONTEXTO     │            │
│         ├─────────────┼─────────────┼─────────────────┤            │
│         │ Quais fontes│ Qual padrão │ O que não está  │            │
│         │ são válidas │ de qualidade│ nos autos       │            │
│         └─────────────┴─────────────┴─────────────────┘            │
│                                                                     │
│         Sem curadoria humana, agentes são máquinas                 │
│         velozes correndo no escuro.                                │
│                                                                     │
│   ─────────────────────────────────────────────────────────────    │
│                                                                     │
│         O Regente também define QUAIS FERRAMENTAS o agente         │
│         pode usar. Cada tool é uma porta. Cada permissão           │
│         concedida é um risco assumido.                             │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

**Exemplo jurídico:**
- **Conhecimento:** Configurar o agente para buscar apenas em bases oficiais (STF, STJ, TRFs), não em blogs jurídicos
- **Valores:** Estabelecer que minutas devem seguir o padrão de fundamentação do seu gabinete
- **Contexto:** Informar ao agente que este autor já teve 3 ações anteriores negadas — dado que não está nos autos atuais
- **Ferramentas:** Definir que o agente pode acessar o PJe mas não pode enviar petições automaticamente

> **Nota técnica:** Em termos do Claude Agent SDK, isso significa definir quais *tools* o agente pode invocar e com quais *permissions*.

---

**Princípio III: Construtor de Cada Peça**

> "Você não apenas rege a orquestra — você a **constrói**."

```
┌─────────────────────────────────────────────────────────────────────┐
│  PRINCÍPIO III                                                      │
│                                                                     │
│         ✓ Cada agente foi moldado por você                         │
│         ✓ Cada skill foi escrito ou adaptado por você              │
│         ✓ Cada prompt foi testado e refinado por você              │
│         ✓ Cada workflow foi desenhado, quebrado e consertado       │
│                                                                     │
│   ─────────────────────────────────────────────────────────────    │
│                                                                     │
│         O que você constrói é SEU.                                  │
│         Ajustado à sua prática. Afinado à sua visão.               │
│         Espelho do seu julgamento.                                  │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

**Exemplo jurídico:**
- Você cria um skill `/analisar-contestacao` que extrai os pontos controvertidos
- Você ajusta o prompt para que ele identifique também prescrição e decadência
- Você testa com 10 casos reais e corrige os erros
- Resultado: uma ferramenta que funciona **do seu jeito**

> **Nota técnica:** Skills são instruções reutilizáveis que você escreve. O agente as executa, mas a autoria é sua.

---

**Princípio IV: A Tentação de Renunciar**

> "Aqui está o perigo que as propagandas não mencionam."

```
┌─────────────────────────────────────────────────────────────────────┐
│  PRINCÍPIO IV                                                       │
│                                                                     │
│         A TENTAÇÃO                                                  │
│         "Eles são tão bons... talvez eu possa deixá-los            │
│          fazer sozinhos. Talvez eu não precise mais revisar."      │
│                                                                     │
│   ─────────────────────────────────────────────────────────────    │
│                                                                     │
│         O humano que renuncia não se liberta.                      │
│         Se torna REFÉM de um sistema que não entende mais.         │
│                                                                     │
│         RESISTA. Permaneça no palco.                               │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

**Exemplo jurídico:**
- O agente produz uma minuta de sentença impecável em 5 casos seguidos
- No 6º caso, ele alucina um precedente que não existe
- Se você parou de revisar, assinou uma sentença com fundamentação falsa
- A assinatura é sua. A responsabilidade é sua.

> O agente é ferramenta. A toga é sua.

**Transição:**
> "Os primeiros quatro princípios definem quem você é. Os próximos três definem **como você opera**."

### Bloco 2: A Arte da Regência — Princípios V-VII (7 min)

---

**Princípio V: O Agente Coordena, Você Projeta**

```
┌─────────────────────────────────────────────────────────────────────┐
│  PRINCÍPIO V                                                        │
│                                                                     │
│         A DIVISÃO FUNDAMENTAL                                       │
│                                                                     │
│         O AGENTE coordena a execução:                               │
│         • Invoca ferramentas (tool use)                             │
│         • Gerencia subtarefas                                       │
│         • Adapta-se a erros e retenta                              │
│                                                                     │
│         VOCÊ define a estratégia:                                   │
│         • Qual problema resolver                                    │
│         • Quais constraints aplicar                                 │
│         • Quando intervir                                           │
│         • O que é sucesso                                           │
│                                                                     │
│   ─────────────────────────────────────────────────────────────    │
│                                                                     │
│         O agente executa a partitura.                               │
│         Você ESCREVE a partitura.                                   │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

**Exemplo jurídico:**
- **Você projeta:** "Quero um workflow que baixe o processo, extraia os fatos, pesquise precedentes e gere um relatório"
- **O agente coordena:** Invoca o tool de download, depois o de extração, depois o de pesquisa, depois o de redação
- **Você avalia:** O relatório atende ao padrão? Os precedentes são pertinentes?

> **Nota técnica:** O agente usa *task coordination* para gerenciar múltiplas chamadas. Mas a arquitetura do workflow é decisão sua.

---

**Princípio VI: Presença Intencional**

```
┌─────────────────────────────────────────────────────────────────────┐
│  PRINCÍPIO VI                                                       │
│                                                                     │
│         O Regente não faz tudo. Mas está presente onde importa.    │
│                                                                     │
│         PRESENTE em:                    AUSENTE em:                 │
│         • Concepção do workflow         • Cada token gerado        │
│         • Avaliação dos resultados      • Cada log de execução     │
│         • Decisões de mudança           • Cada arquivo temporário  │
│         • Evolução do sistema           • Engrenagens internas     │
│                                                                     │
│   ─────────────────────────────────────────────────────────────    │
│                                                                     │
│         O equilíbrio: saber onde estar, por que estar,             │
│         e quando a ausência é confiança vs negligência.             │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

**Exemplo jurídico:**
- **Presença necessária:** Revisar a minuta final antes de assinar
- **Ausência adequada:** Não precisa acompanhar cada página sendo convertida de PDF para texto
- **Negligência:** Nunca verificar se os precedentes citados existem

> Escolha seus pontos de intervenção. Nem em tudo, nem em nada.

---

**Princípio VII: A Humildade que Liberta**

> "Sua pesquisa encontrou TODOS os precedentes relevantes — ou apenas os que apareceram nas primeiras tentativas, com os termos que você pensou, no tempo que restava?"

```
┌─────────────────────────────────────────────────────────────────────┐
│  PRINCÍPIO VII                                                      │
│                                                                     │
│         LIMITAÇÕES HUMANAS:                                         │
│         • Cansaço na página 300                                     │
│         • Viés de confirmação nas buscas                            │
│         • Esquecimento de cruzar dados distantes                   │
│                                                                     │
│         LIMITAÇÕES DO AGENTE:                                       │
│         • Pode alucinar informações                                 │
│         • Pode se confundir com documentos muito grandes            │
│         • Pode perder nuances que o contexto não captura           │
│                                                                     │
│   ─────────────────────────────────────────────────────────────    │
│                                                                     │
│         NÃO É SUBSTITUIÇÃO. É COMPLEMENTARIDADE.                    │
│                                                                     │
│         O agente ilumina seus pontos cegos.                         │
│         Você ilumina os pontos cegos do agente.                     │
│                                                                     │
│   ─────────────────────────────────────────────────────────────    │
│                                                                     │
│         O orgulho sussurra: "Eu dou conta sozinho."                │
│         A sabedoria responde: "Juntos, entregamos mais."           │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

**Exemplo jurídico:**
- **O agente ajuda você:** Encontra um precedente de 2018 que você não lembraria de buscar
- **Você ajuda o agente:** Percebe que ele confundiu "auxílio-doença" com "auxílio-acidente" — erro sutil que exige conhecimento jurídico

> Humildade dupla: reconhecer seus limites E os limites do agente.

**Transição:**
> "Esses sete princípios são a base. Os três últimos são a **disciplina** que transforma conhecimento em maestria."

**Princípio VIII: A Simplicidade como Disciplina**

```
┌─────────────────────────────────────────────────────────────────────┐
│  PRINCÍPIO VIII                                                     │
│                                                                     │
│         A TENTAÇÃO DA COMPLEXIDADE                                  │
│                                                                     │
│         Você aprende que pode criar workflows multi-etapa,          │
│         sistemas multi-agente, orquestrações elaboradas...         │

│         E então quer usar TUDO.                                     │
│                                                                     │
│   ─────────────────────────────────────────────────────────────    │
│                                                                     │
│         A LIÇÃO DA ANTHROPIC:                                       │
│         "Encontre a solução mais simples possível,                  │
│          aumentando complexidade apenas quando necessário."         │
│                                                                     │
│   ─────────────────────────────────────────────────────────────    │
│                                                                     │
│         Complexidade é CUSTO, não benefício.                        │
│         A elegância está na simplicidade que funciona.              │
│                                                                     │
│         Pergunta-chave: "Isso é necessário, ou é vaidade técnica?" │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

**Exemplo jurídico:**
- **Complexo demais:** 5 agentes em paralelo, cada um analisando um aspecto do processo, com um orquestrador consolidando
- **Simples e eficaz:** Um único agente com um bom prompt que analisa o processo sequencialmente
- **Regra:** Comece simples. Adicione complexidade só quando o simples falhar.

> **Nota técnica:** A documentação da Anthropic recomenda explicitamente começar com *single-agent workflows* antes de escalar para *multi-agent systems*.

**Exemplo jurídico:**

> **Nota técnica:** A *context window* tem limite fixo. Informação demais = atenção diluída = resultados piores.

## Notas para o Instrutor

1. **Conexão com Aula 0:** A abertura assume que o aluno já viu as 7 limitações e as 6 vantagens. Não repetir.
2. **Ritmo:** Cada princípio deve ter ~1 minuto de exposição + exemplo + pausa para absorção.
3. **Exemplos:** Use os exemplos jurídicos fornecidos ou substitua por casos da sua experiência.
4. **Interação:** Após cada bloco de princípios, perguntar: "Algum desses já é familiar? Algum parece impossível?"
5. **Notas técnicas:** Mencione brevemente para mostrar rigor, mas não aprofunde — isso vem nas próximas aulas.
6. **Foco:** O core desta aula são os PRINCÍPIOS. Não desviar para técnica.
7. **Fechamento:** Garantir que todos identifiquem seu princípio-âncora antes de dispensar.
