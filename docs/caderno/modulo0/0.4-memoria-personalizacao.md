# 0.4 Memória e Personalização

## Objetivo
Entender os recursos de memória e personalização do Claude — e por que NÃO usá-los de forma global.

## O que é Memória?

A **memória** é um recurso que permite ao Claude lembrar informações de conversas anteriores. Quando ativada, o modelo:

- Extrai "lembranças" das suas interações
- Usa essas lembranças para contextualizar conversas futuras
- Aprende seus padrões, preferências e estilo

**Exemplo:** Se você menciona que é juiz federal em uma conversa, o Claude pode lembrar disso em conversas futuras e adaptar as respostas.

## O que é Personalização?

A **personalização** permite definir instruções que valem para **todas** as conversas. É um texto fixo que o Claude sempre considera antes de responder.

**Exemplo:** Você pode configurar: "Sempre responda em português formal" ou "Sou advogado tributarista, adapte as respostas para esse contexto".

## Por que NÃO Usar (Abordagem Minimalista)

Para uso profissional diversificado, recomendamos **desativar** memória e **não usar** personalização global. Motivos:

### Problema 1: Contaminação de Contexto

| Situação | Risco |
|----------|-------|
| Você analisa um caso de família pela manhã | A memória registra termos e conceitos |
| À tarde, analisa um caso tributário | O Claude pode "contaminar" a análise com contexto do caso anterior |

A memória não distingue bem entre contextos diferentes. O que foi útil em um caso pode ser ruído em outro.

### Problema 2: Estilo Único Não Serve

No trabalho jurídico, você precisa de estilos diferentes:

| Tarefa | Estilo Necessário |
|--------|-------------------|
| Relatório | Neutro, descritivo |
| Fundamentação | Argumentativo, técnico |
| Despacho | Direto, objetivo |
| Análise crítica | Questionador, profundo |

Se você personaliza um estilo global, ele vai "vazar" para contextos onde não é adequado.

### Problema 3: Técnicas Melhores Existem

Em vez de personalização global, você aprenderá no Módulo 2:

- **Persona por prompt** — Define o papel específico para cada tarefa
- **Projetos** — Contexto isolado por tipo de trabalho
- **System prompts** — Instruções precisas por conversa
- **Templates POEMA** — Estrutura completa de customização

Essas técnicas são mais **precisas** e **controláveis** que a personalização global.

## Onde Ficam Essas Configurações?

### Memória
1. Perfil (canto inferior esquerdo) → Configurações
2. Menu lateral → **Privacidade**
3. Seção **"Preferências de memória"** → Gerenciar

### Personalização
1. Perfil → Configurações
2. Menu lateral → **Geral**
3. Campo **"Quais preferências pessoais o Claude deve considerar?"**

## Exercício: Verificar e Desativar

### O que você vai fazer
Verificar se memória e personalização estão desativadas.

### Passo a passo

**1. Verificar Memória:**
- Acesse: Configurações → Privacidade
- Clique em **"Preferências de memória"** → Gerenciar
- Se houver lembranças salvas, avalie se quer mantê-las
- **Recomendação:** Limpar lembranças existentes

**2. Verificar Personalização:**
- Acesse: Configurações → Geral
- Localize o campo de preferências pessoais
- **Recomendação:** Deixar em branco

**3. Confirmar configuração minimalista:**
- Memória: vazia ou desativada
- Personalização: campo em branco

### Checklist de Verificação
- [ ] Acessei as configurações de memória
- [ ] Entendo o que a memória faz
- [ ] Memória está limpa ou desativada
- [ ] Campo de personalização está em branco
- [ ] Entendo por que não usar personalização global

## Quando USAR Memória/Personalização?

Existem casos específicos onde pode fazer sentido:

| Cenário | Usar? | Por quê |
|---------|-------|---------|
| Uso pessoal casual | Pode | Não há risco de contaminação profissional |
| Um único tipo de trabalho sempre | Pode | Contexto consistente |
| Trabalho jurídico variado | **Não** | Risco de contaminação |
| Múltiplos estilos necessários | **Não** | Estilo único atrapalha |

## A Alternativa: Projetos

Em vez de personalização global, use **Projetos** para isolar contextos:

```
┌─────────────────────────────────────────────────────────────┐
│                    SEM PROJETOS (ruim)                      │
│                                                             │
│   Memória Global ──► Todas as conversas ──► Contaminação    │
│                                                             │
└─────────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│                    COM PROJETOS (bom)                       │
│                                                             │
│   Projeto "Análise FIRAC"  ──► Conversas de análise         │
│   Projeto "Relatórios"     ──► Conversas de relatório       │
│   Projeto "Pesquisa"       ──► Conversas de pesquisa        │
│                                                             │
│   Cada projeto com suas próprias instruções, isolado.       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

Você aprenderá a criar e usar Projetos de forma avançada no Módulo 2.

## Dica Prática
> Pense na memória como um "assistente que anota tudo". Parece útil, mas em contexto profissional, você não quer que anotações de um cliente vazem para outro. Melhor começar cada conversa com contexto explícito e controlado.

## Armadilha Comum
> **"Vou configurar minha personalização como 'Sou juiz federal, responda de forma técnica'."**
>
> Parece eficiente, mas causa problemas:
> - Quando você quiser um texto mais acessível, o Claude vai insistir no técnico
> - Quando quiser explorar argumentos do autor, o Claude vai manter a perspectiva de julgador
> - Você perde flexibilidade
>
> **Solução:** Defina a persona em cada prompt, não globalmente.
